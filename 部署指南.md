# 手把手：把这个网站发布到 Vercel

> 我（AI）无法用你的账号帮你登录/上传，所以这一步需要你在**自己的 Mac 上**操作。跟着做，大约 10 分钟。全程遇到任何一步卡住，把屏幕截图发我。

下面有两种方法，**推荐方法 A（最简单，5 条命令，不需要 GitHub）**。方法 B 稍复杂但以后每次更新会自动上线。

---

## 方法 A · Vercel 命令行（推荐，最快）

### 第 1 步：确认电脑装了 Node
1. 按 `⌘ + 空格`，输入 **Terminal（终端）**，回车打开。
2. 在终端里粘贴这行，回车：
   ```bash
   node -v
   ```
   - 如果显示类似 `v20.x.x` → 已装好，跳到第 2 步。
   - 如果显示 `command not found` → 去 https://nodejs.org 下载 **LTS** 版本，双击安装（一路"继续"），装完**关掉终端重新打开**，再试一次。

### 第 2 步：进入这个网站文件夹
在终端里输入 `cd ` （**cd 后面有一个空格**），然后把 **Pharma curriculum 这个文件夹**从访达直接**拖进终端窗口**（会自动填上路径），回车：
```bash
cd /Users/mini/Routine\ AI\ Tools/Pharma\ curriculum
```
（拖拽会自动生成正确路径，不用手打。）

（可选）清掉生成环境留下的半成品 git 文件夹，保持干净：
```bash
rm -rf .git .vercel
```

### 第 3 步：安装 Vercel 工具
```bash
npm i -g vercel
```
等它跑完（出现新的一行光标即可）。若报权限错误，改用：`sudo npm i -g vercel`（会让你输开机密码，输入时看不见字符是正常的）。

### 第 4 步：登录 Vercel
```bash
vercel login
```
- 用方向键选 **Continue with GitHub** 或 **Continue with Email** 都行，回车。
- 选 Email 的话：输入你的邮箱 → 去邮箱点验证邮件里的按钮 → 回到终端会自动显示登录成功。
- 没有 Vercel 账号也没关系，这一步会顺带帮你注册。

### 第 5 步：发布上线
```bash
vercel --prod
```
它会问你几个问题，**基本一路回车用默认**即可：

| 它问什么 | 你怎么答 |
|---|---|
| Set up and deploy? | 回车（Y） |
| Which scope? | 回车（选你自己的账号） |
| Link to existing project? | 回车（N，新建） |
| What's your project's name? | 回车，或输入 `pharma-curriculum` |
| In which directory is your code? | 回车（就是当前 `./`） |
| Want to modify settings? | 输入 `n` 回车 |

跑完后，终端会打印一行 **Production: https://xxxx.vercel.app** —— 那就是你的网址，随时能打开、能分享。🎉

### 以后想更新网站？
每当我（AI）帮你把词典又加了内容，你只需在这个文件夹里再跑一次：
```bash
vercel --prod
```
几秒后同一个网址就更新了。

## 方法 B · GitHub 自动部署（进阶：以后 push 就自动上线）

适合你想"改完自动更新、不用每次敲命令"。需要一个 GitHub 账号。

1. 装 **GitHub Desktop**：https://desktop.github.com （图形界面，不用命令行）。
2. 先删掉本文件夹里的旧 `.git`（方法 A 第 2 步那条 `rm -rf .git`），避免冲突。
3. GitHub Desktop → **File → Add Local Repository** → 选这个文件夹 → 它会提示"create a repository" → 点创建 → 再点 **Publish repository**（可勾选 Private）。
4. 打开 https://vercel.com → 用 GitHub 登录 → **Add New → Project → Import** 刚才那个仓库 → Framework 选 **Other**（纯静态，无需 build）→ **Deploy**。
5. 以后在 GitHub Desktop 里点 **Commit → Push**，Vercel 会自动重新部署。

## 常见问题
- **`vercel: command not found`**：第 3 步没装成功，重跑 `npm i -g vercel`；或关掉终端重开。
- **打开网址是 404**：确认发布时选的目录是文件夹根目录（里面能看到 `index.html`）。本项目 `index.html` 就在根目录，正常不会 404。
- **想换更好记的网址**：Vercel 项目 → Settings → Domains，可改成 `你起的名字.vercel.app` 或绑定自有域名。
- **担心别人看到**：方法 B 发布时把仓库设为 Private；Vercel 部署默认是公开可访问的链接（不主动分享一般搜不到），需要更严格的访问控制可在 Vercel 加 Password Protection（付费）。
